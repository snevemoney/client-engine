# Operator Framework — How to Use and Build Around Your Strengths

**Core idea:** Don't try to be every role. Know your top 1–3 strengths. Build systems + people around your weak spots. Position yourself correctly. Use habits/checklists so execution becomes automatic.

This doc maps the Pat/Tom-style framework onto your app. Aligns with [CLIENT_ENGINE_AXIOMS.md](./CLIENT_ENGINE_AXIOMS.md) and [PROJECT_CONTEXT.md](../PROJECT_CONTEXT.md).

## 9. Weekly Operator Habits (Checklist)

**Daily (5 min):**

- [ ] Open Command Center — scan today's AI activity
- [ ] Check lead queue — anything needs follow-up?
- [ ] Meta Ads (if configured) — KPI trends, critical recs

**Weekly (15 min):**

- [ ] Run full operator loop (see PROD_OPERATOR_LOOP.md)
- [ ] Review Sales Leak — where are leads stalling?
- [ ] Check Metrics — pipeline success rate, recent failures
- [ ] One deploy or one doc update (runbook, checklist)

**Monthly (30 min):**

- [ ] Proof audit — what can you show? Before/after? Time saved?
- [ ] Positioning check — still accurate? Still tight?
- [ ] Talent check — which bottleneck needs human help first?

## 10. Tom's Practical Questions (Use Regularly)

Before adding features or doing more work:

1. **Are you using this to make money now?** — Not "someday." Now.
2. **Can a client understand what this does in 30 seconds?** — If not, simplify presentation.
3. **Do you have a simple offer tied to this app?** — "I build X for Y in Z time."
4. **Can this create proof fast?** — Before/after, saved time, fewer errors, lead response speed.
5. **Are you shipping, or polishing forever?** — Classic builder trap.
